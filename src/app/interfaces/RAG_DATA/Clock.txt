# Clock API

## Endpoint Information
- **Method:** GET
- **URL:** https://api.finam.ru/v1/assets/clock
- **Description:** Получение времени на сервере

## Request Parameters
Нет параметров запроса

## Request Headers
- **Authorization:** JWT-токен (обязательно)

## Response Parameters
- **timestamp** (string): Метка времени сервера в формате ISO 8601

## Example Response
```json
{
  "timestamp": "2025-07-24T08:05:50.966209637Z"
}
```

## Key Concepts

### Server Time
- **timestamp**: Текущее время сервера Finam в UTC
- **Format**: ISO 8601 с микросекундной точностью
- **Timezone**: UTC (время по Гринвичу)

### Time Format Details
- **Format**: `YYYY-MM-DDTHH:MM:SS.ffffffZ`
- **YYYY**: Год (4 цифры)
- **MM**: Месяц (01-12)
- **DD**: День (01-31)
- **T**: Разделитель даты и времени
- **HH**: Час (00-23)
- **MM**: Минуты (00-59)
- **SS**: Секунды (00-59)
- **ffffff**: Микросекунды (6 цифр)
- **Z**: Индикатор UTC времени

## Usage Examples

### Basic Request
```
GET /v1/assets/clock
Authorization: Bearer your_jwt_token
```

### Response Processing
```python
import datetime

# Получение времени сервера
response = requests.get(
    "https://api.finam.ru/v1/assets/clock",
    headers={"Authorization": jwt_token}
)
server_time = response.json()["timestamp"]

# Парсинг времени
server_datetime = datetime.datetime.fromisoformat(server_time.replace('Z', '+00:00'))
print(f"Время сервера: {server_datetime}")

# Сравнение с локальным временем
local_time = datetime.datetime.now(datetime.timezone.utc)
time_diff = server_datetime - local_time
print(f"Разница с локальным временем: {time_diff}")
```

## Use Cases

### Synchronization
- Синхронизация времени клиента с сервером
- Корректировка локальных часов
- Устранение расхождений во времени

### Trading Operations
- Проверка времени перед торговыми операциями
- Валидация временных меток в запросах
- Определение рабочего времени бирж

### Logging and Debugging
- Синхронизация логов с серверным временем
- Отладка временных проблем
- Анализ задержек в API запросах

## Usage Notes
- Обязательно передавать JWT-токен в заголовке Authorization
- Время возвращается в UTC (всемирное координированное время)
- Точность до микросекунд
- Используйте для синхронизации клиентского времени
- Полезно для определения актуальности данных
- Рекомендуется периодически обновлять время сервера
